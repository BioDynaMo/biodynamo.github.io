◆ CriticalRegion()

virtual void bdm::SimObject::CriticalRegion ( std::vector< Spinlock *> *  locks)
inlinevirtual

If the thread-safety mechanism is set to user-specified this function will be called before the operations are executed for this simulation object.
Subclasses define the critical region by adding the locks of all simulation objects that must not be processed in parallel.
Don't forget to add the lock of the current simulation object.

See also
Param::thread_safety_mechanism_
